# 题目
搜索旋转排序数组

# 题目理解
一个原本升序的数组在某个位置旋转过，比如：
原数组：[0,1,2,4,5,6,7]
旋转后：[4,5,6,7,0,1,2]
要求用 O(log n) 复杂度查找 target 的索引。
“旋转排序数组”——原本有序，在某个点断开后交换了前后两部分


# 思路
1. 虽然整个数组不是有序的，但二分之后，左半和右半必有一半是有序的。
2. 判断哪一半有序：如果 nums[left] <= nums[mid]，则左半有序；否则右半有序。
3. 在有序的那一半里，判断 target 是否在范围内：
   - 如果在，就去有序那一半继续二分。
   - 如果不在，就去另一半继续二分。
4. 每次循环砍掉一半搜索空间，所以是 O(log n)。

# 关键代码片段
```cpp
if (v[left] <= v[mid]) {      // 左半有序
    if (v[left] <= x && x < v[mid])
        right = mid - 1;       // 在左半
    else
        left = mid + 1;        // 去右半
} else {                       // 右半有序
    if (v[mid] < x && x <= v[right])
        left = mid + 1;        // 在右半
    else
        right = mid - 1;       // 去左半
}
